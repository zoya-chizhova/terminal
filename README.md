# terminal
Запуск программы:\
#Компиляция:\
g++ -o terminal main.cpp Terminal.cpp Entry.cpp Directory.cpp SingleLineFile.cpp MultiLineFile.cpp
#Запуск файла:
в Linux/macOS: ./terminal;
в Windows: terminal.exe.

После запуска появится приглашение терминала:

Virtual Terminal started. Type 'help' for commands.
terminal>
Вводите команды и нажимайте Enter для их выполнения.

Поддерживаемые команды:
help -	Показать список доступных команд	\
exit	- Завершить работу программы	
ls	- Показать содержимое текущей директории
cd <dir> -	Перейти в директорию <dir>; cd .. — подняться на уровень выше	
cat <file> -	Показать содержимое файла <file>	
touch <file> -	Создать новый однострочный файл <file> в текущей директории	
mkdir <dir>	- Создать новую директорию <dir> в текущей директории	
rm <entry> -	Скрыть (не удалить физически) файл или директорию 
restore <entry> -	Восстановить ранее скрытую сущность 
mv <old> <new>	- Переименовать сущность <old> в <new>	
edit <file> <content>	- Изменить содержимое файла <file> на <content>. Поведение зависит от типа файла	
##Архитектурные решения
1. Иерархия классов
Корневой класс Entry. Базовый класс для всех сущностей файловой системы. Содержит общие поля (name, hidden) и виртуальные методы.

Наследование:

Directory наследуется от Entry.

SingleLineFile наследуется от Entry.

MultiLineFile наследуется от Entry.

2. Полиморфизм и виртуальные методы
Ключевые виртуальные методы в Entry:

displayContent() — выводит содержимое сущности. У Directory выводит список видимых дочерних элементов, у файлов — их содержимое.

edit(const std::string&) — изменяет содержимое. У директорий выводит ошибку, у SingleLineFile и MultiLineFile реализует свою логику.

asDirectory() — возвращает указатель на себя, если это Directory, иначе nullptr. Позволяет безопасно проверять тип объекта без dynamic_cast.

3. Модель файловой системы
Дерево объектов. Корень — объект root типа Directory. Текущая директория (currentDir) — указатель на объект Directory.

Хранение дочерних элементов. В классе Directory есть вектор children с умными указателями std::unique_ptr<Entry>. Это обеспечивает автоматическое управление памятью и предотвращает утечки.

Текущий рабочий каталог (CWD). Реализован как указатель currentDir на объект Directory. Переход (cd) — смена значения указателя.

4. Работа со скрытыми сущностями
Флаг скрытости. Поле bool hidden в классе Entry. rm устанавливает hidden = true, restore — hidden = false.

Фильтрация при ls. Реализована внутри метода displayContent() класса Directory: перебираются все дочерние элементы, но выводятся только те, у которых !isHidden().

5. Поиск сущностей
Локализация: метод findChild(const std::string& name) в классе Directory. Ищет дочерний элемент по имени с помощью std::find_if.

Использование: вызывается в Terminal при обработке команд cd, cat, rm, restore, mv.

6. Типы файлов
SingleLineFile. Хранит содержимое в одной строке (std::string content). Команда edit перезаписывает строку.

MultiLineFile. Хранит содержимое как вектор строк (std::vector<std::string> lines). В текущей реализации edit заменяет все строки на одну новую.


7. Обработка команд
Центральный цикл в Terminal::run(). Бесконечный цикл чтения ввода и вызова processCommand().

Разбор команд. Используется std::istringstream для разбиения строки ввода на токены (команда и аргументы).

Диспетчеризация. Большая цепочка if-else if по первой команде. Внутри каждой ветки вызываются соответствующие методы currentDir или Terminal.
